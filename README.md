# AI-POWERED DIGITAL MEDIA AUTHENTICATION AND VERIFICATION SYSTEM

## Project Overview 

This project is a public fake-media verification system that helps users check whether digital content is trustworthy before sharing it. The service analyzes image, video, audio, and social-post context, then returns a clear decision with confidence and explanation.

### What Problem We Are Solving

Users frequently encounter suspicious content online but usually do not have a fast and reliable way to verify authenticity. Because of this, fake media is often shared unknowingly.

### What Our Project Does

- Accepts user-submitted media (image/video/audio) for authenticity analysis.
- Optionally analyzes social-post context (platform, post text, timestamp, claimed location).
- Runs multiple forensic checks and combines them into one final assessment.
- Returns a decision with confidence, indicators, and a report path for traceability.

### How It Works (End-to-End)

1. User uploads media or submits social post context.
2. The system runs specialized forensic modules (metadata, pixel/audio/video, deepfake checks, cross-modal consistency).
3. A fusion engine combines module scores into one final result.
4. The API returns classification, confidence signals, and key indicators.
5. The result is stored in the configured database and can be shown in the dashboard.

### Output Meaning for Non-Technical Reviewers

- `authentic`: Content appears genuine.
- `ai_generated`: Content likely generated by AI.
- `manipulated`: Content likely edited/tampered.
- `inconclusive`: Not enough evidence for a reliable final decision.

## Capabilities

- Metadata forensics: EXIF, parser metadata, software signatures, timestamp consistency.
- Pixel forensics: ELA, JPEG artifact blockiness, residual noise profile, lighting inconsistency.
- Deepfake model layer: PyTorch CNN inference interface with pluggable checkpoints.
- Video temporal forensics: frame transition and interpolation anomaly scoring.
- Audio forensics: MFCC, spectral flatness, pitch contour-derived indicators.
- Cross-modal consistency: audio-video confidence mismatch checks.
- Evidence integrity: SHA-256 hashing for chain-of-custody integrity.
- Confidence fusion engine: weighted aggregation into final authenticity assessment.
- Legal report output: structured PDF report.
